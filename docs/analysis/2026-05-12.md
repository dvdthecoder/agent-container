## Key findings

### 1. Conventions don't pay off on simple, well-scoped tasks

The frugal injection principle (lesson #25) states that injected context must save more tokens
than it adds. On this task, conventions added **+334 prompt tokens** (1.3%) and saved **zero**
— turn count was identical (4 turns, `read → edit` in both conditions).

**Why:** The task string already told the agent exactly what file to touch and what to fix.
The fixture repo has 3 files. There was nothing to explore. Conventions are inert when the
agent has no exploratory work to skip.

### 2. Conventions are valuable when the agent would otherwise explore

The value of `AGENTS.md` / `AgentTaskSpec.conventions` scales with exploratory overhead:

| Scenario | Agent's first moves (no conventions) | Conventions save |
|---|---|---|
| 3-file repo, explicit task | `read mathlib.py` → `edit` | nothing |
| 50-file repo, vague task | `glob **/*.py` → `read` × 3–5 → `edit` | 3–5 turns (~15–25k tokens) |
| Multi-component task | `glob` + `read` + `bash grep` × N | N turns |

Conventions should carry: repo map, which files are relevant, test command, constraints.
They are not useful when the task string already encodes all of this.

### 3. Success rate hint (inconclusive at N=2)

Without conventions: 1/2 succeeded. The failure on run 2 was the end_turn race — `think_chars=1`
is a parsing artefact from a single stray character, not real thinking. The retry fix (#152)
reduced the flake but didn't eliminate it on 32B. More data needed.

With conventions: 2/2 succeeded. Possibly conventions help the model produce a cleaner
`end_turn` signal by giving it a clearer definition of done (acceptance criteria). Inconclusive
at 2 runs.

### 4. Conventions overhead is negligible when inert

+334 tokens on a ~26k-token prompt is 1.3%. Even when conventions don't help, they don't hurt
meaningfully on cost. The risk is not token waste — it's cognitive noise for the model when
the conventions are irrelevant to the task.

---

## Revised frugal injection guidance (update to lesson #25)

**Before:** "Measure before injecting. Speculative injection is waste."

**After:** Conventions pay off proportionally to exploratory overhead they eliminate:
- Short task + tiny repo + explicit file reference → **skip conventions**, they add noise
- Vague task + large repo + no file guidance → **inject conventions**, they replace exploration
- Any task with non-obvious test command → **always inject** the test command line (~3k tokens saved per discovery turn)

The `acceptance_criteria` field in `AgentTaskSpec.resolved_prompt()` is the highest-value
injection: it tells the agent when it's done without requiring a test-discovery turn.
