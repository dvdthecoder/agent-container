## Key findings

### 1. `<think>` stripping reduces Qwen3 prompt tokens by 27%

| Date | Qwen3 avg prompt tok | Change |
|---|---|---|
| 2026-05-08 (baseline, no stripping) | 46,149 | — |
| 2026-05-09 (`<think>` stripping active) | 33,888 | **−27%** |

Qwen3 thinking tokens (~1,000–2,400 per turn) were accumulating in the conversation history and
being resent as context on every subsequent turn. Stripping `<think>…</think>` blocks from
assistant messages before they re-enter the opencode session breaks the compounding.

**Important:** the completion token count for Qwen3 (1,056) is still higher than Coder models
(117–274). vLLM counts thinking tokens in completion regardless of stripping — the model still
*generates* them. The stripping only prevents them from *re-entering the context window* on
subsequent turns. For single-turn tasks the saving is zero; for multi-turn runs it prevents
exponential growth.

### 2. opencode/aider prompt ratio: 8.4× (down from 9.6×)

| Date | opencode prompt | aider prompt | Ratio |
|---|---|---|---|
| 2026-05-05 (baseline) | ~31,000 | ~2,600 | 11.9× |
| 2026-05-08 (post description-strip) | 26,034 | — | — |
| 2026-05-09 (post description-strip, head-to-head) | 27,573 | 3,300 | **8.4×** |

The improvement from 11.9× to 8.4× over two iterations came from:
- Stripping per-property `description` fields from tool schemas (−19% prompt tokens, #150)
- Post-edit tool filtering: after the first edit/write, the 10-tool schema is replaced by
  the 4-tool verification set (bash, read, grep, glob)

The remaining 8.4× gap is irreducible framework overhead from the Responses API adapter:
opencode resends the full conversation history on every turn, and the session preamble
adds ~2k tokens unavoidable overhead. aider sends only the task + changed files.

### 3. 32B reliability: 2/2 ✅

On 2026-05-08, `qwen2.5-coder-32b` / opencode failed 2/3 runs (empty diff, end_turn race
condition, lesson #19). Today: 2/2 succeeded (NO_PR=1, ~120s each).

Two runs is not enough to declare the race fixed — the flake rate was ~33% previously so
two consecutive successes is within normal variance. #152 (grace period fix) remains open.
More data needed before closing.
